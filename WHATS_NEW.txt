v14 — 24 June 2026  [VERCEL MIGRATION]
------------------------------------------------------------
PLATFORM MIGRATION — Netlify → Vercel
- Migrated from Netlify to Vercel hosting
- Vercel free tier has no function invocation limits
  (vs Netlify's 125,000/month limit that was exhausted)
- All 24 serverless functions migrated to Vercel api/ format
- vercel.json replaces netlify.toml for routing config
- All HTML, CSS, JS, Supabase database UNCHANGED
- netlify/functions/ kept in zip for reference but
  Vercel uses api/ directory instead

SETUP STEPS FOR VERCEL (see below)
1. Create free account at vercel.com
2. Install Vercel CLI or use dashboard drag & drop
3. Add all environment variables (same as Netlify)
4. Update Paystack webhook URL to new Vercel domain
5. Update Google OAuth redirect URI if needed

ENVIRONMENT VARIABLES NEEDED IN VERCEL:
  SUPABASE_URL
  SUPABASE_ANON_KEY
  SUPABASE_SERVICE_KEY
  ADMIN_SECRET
  PAYSTACK_SECRET_KEY
  PAYSTACK_WEBHOOK_SECRET
  VAPID_PUBLIC_KEY
  VAPID_PRIVATE_KEY

UPDATE AFTER DEPLOYING:
- Paystack webhook URL → https://YOUR-VERCEL-URL/api/paystack-webhook
- Google OAuth redirect → https://YOUR-VERCEL-URL/auth-callback.html
  (add this in Google Cloud Console → Credentials)
- Supabase redirect URL → add YOUR-VERCEL-URL in
  Authentication → URL Configuration

v13.6 — 24 June 2026
------------------------------------------------------------
CRITICAL FIX — SERVICE WORKER CACHE BREAKING SITE
- The v13.3 service worker was caching all site files
  aggressively on install — when a cached version had
  a bug, the service worker kept serving the broken
  version to all users even after the site was updated
- sw.js completely rebuilt in PASSIVE mode:
    → No caching on install
    → Deletes ALL existing caches on activate  
    → Never intercepts fetch requests
    → Only handles push notifications (its real job)
- This means the site always loads fresh from Netlify
- Users affected by the broken cache will be fixed
  automatically when their browser downloads this new
  service worker

ACTION REQUIRED FOR USERS STILL SEEING BROKEN SITE:
1. Open DevTools (F12) → Application tab
2. Click Service Workers → Unregister
3. Click Storage → Clear site data
4. Refresh the page

------------------------------------------------------------
v13.5 — 24 June 2026
------------------------------------------------------------
FIX — HOLDINGS NOT LOADING (ROOT CAUSE)
- Root cause identified: the holdings table in Supabase
  had no RLS (Row Level Security) policies defined, so
  Supabase blocked all reads and returns empty data
- Fix: run supabase-fix-holdings-rls.sql in Supabase
  SQL Editor — this creates the missing policy that
  allows each user to read their own holdings
- Also fixes portfolio_history table with same issue

IMPROVED
- Holdings now shows a clear error message if loading
  fails instead of hanging on "Loading your holdings…"
- Retry button appears if network error occurs
- Loading spinner shows while holdings are being fetched

SETUP REQUIRED ← IMPORTANT
- Run supabase-fix-holdings-rls.sql in Supabase SQL
  Editor BEFORE deploying this version. Without this
  the holdings will still not load.

------------------------------------------------------------
v13.4 — 25 June 2026
------------------------------------------------------------
FIX — DASHBOARD HOLDINGS NOT LOADING
- Root cause: the PWA service worker script was injected
  inside the print HTML template literal string, breaking
  the JavaScript on the entire dashboard page
- This caused the whole init() function to never run,
  leaving holdings stuck on "Loading your holdings..."
  and leaking raw code (printWin.document.close()) at
  the bottom of the visible page
- Print function template literal now closes correctly
  before printWin.document.close()
- SW registration script added as a separate clean block
  after the main script, not inside any string

  dashboard page

------------------------------------------------------------
FIX — PAYSTACK PREMIUM ACTIVATION

  correctly using PAYSTACK_SECRET_KEY (same key used

  immediately without requiring a logout/login

v13.2 — 24 June 2026
------------------------------------------------------------
FIX — PAYSTACK PREMIUM ACTIVATION
- Root cause found and fixed: webhook signature was being
  computed on re-serialised JSON instead of the raw request
  body string — this caused HMAC mismatch and Paystack
  rejected every webhook silently
- paystack-webhook.js now hashes event.body (raw string)
  exactly as Paystack documentation requires
- paystack-verify.js now ALSO activates Premium directly
  as a fallback — so even if the webhook is delayed or
  fails, Premium activates the moment the user lands on
  the payment success page
- payment-success.html retries verification up to 6 times
  every 5 seconds (handles Mobile Money delay gracefully)
- Cached profile updated immediately in localStorage so
  Premium features unlock without needing to log out/in
- Clear error state shown if activation fails after all
  retries, with contact email for manual resolution

NEW — PRINT PORTFOLIO REPORT
- 🖨 Print button added to the holdings panel header
- Opens a clean, professionally formatted print page with:
    → EagleEyE header and investor name
    → Summary boxes: Total value, Cost basis, Total P&L,
      Day P&L
    → Full holdings table with shares, avg cost, live
      price, value, P&L % and P&L in GH₵ per stock
    → Colour-coded gains (green) and losses (red)
    → Totals row at the bottom
    → Disclaimer footer
- Triggers browser print dialog automatically on open
- Works as both printed document and PDF (Save as PDF)

v13.1 — 24 June 2026
------------------------------------------------------------
NEW — 14-DAY FREE TRIAL
- All new signups automatically get 14 days of full
  Premium access — no card required
- Trial starts the moment they create an account
- Signup flow simplified: plan selection step removed,
  goes straight to account creation in 2 steps
- Signup success screen now highlights the free trial
  with a list of Premium features unlocked
- Trial countdown banner appears on all pages when
  7 or fewer days remain (avoids banner fatigue)
- Banner turns red and says "trial ended" when expired
- After trial expires, premium features lock behind
  the upgrade wall until user pays
- isPremium() in premium-gate.js checks both plan=premium
  AND active trial — one function controls everything
- trial_ends_at returned in login and get-user responses
  so every page can check trial status client-side

SETUP REQUIRED
- Run supabase-update-v4.sql in Supabase SQL Editor
  (adds trial_ends_at column to profiles table)
- NOTE: existing users are backfilled with trial_ends_at
  = now() so they don't get a surprise trial. If you
  WANT existing users to also get a trial, remove the
  UPDATE line from supabase-update-v4.sql before running.
